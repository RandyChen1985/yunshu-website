Enterprise Data & Agent Platform

云枢产品矩阵

以数据服务平台作为治理底座，以智能体平台承接推理、工具与业务执行，再用 OpenClaw Buddy 降低多渠道 AI Agent 的运营门槛。

了解体系 浏览产品
Data API 数据服务化与治理 AI Agent 智能体编排与执行 Buddy OpenClaw 可视化伴侣

Positioning

把企业数据、AI 执行与运维入口放在同一条产品线上。

这不是三个孤立项目的陈列，而是一条从数据供给到智能执行、再到低门槛管理的产品链路。 数据平台负责把物理表、SQL 与 API 资源治理起来；智能体平台负责让 ChatBI、知识库、MCP 工具与企业系统协同运行；OpenClaw Buddy 则把开放式 Agent 网关变成非技术人员也能使用的 Web 工作台。

Product Matrix

三层产品，覆盖从数据到智能体运营的完整闭环。

01

云枢 · 数据服务平台

企业级 Data API 与元数据治理平台，将物理表、自定义 SQL 与语义元数据封装为可审计、 可授权、可观测的 RESTful API。

  • TABLE 零代码映射与 SQL 模板封装
  • SQL Lab、数据产品目录、资产全景
  • RBAC、审计日志、脱敏与 SQL 护栏
GitHub 仓库 ↗

02

云枢 · 智能体平台

面向企业复杂场景的 AI 智能中枢，负责智能路由、AgentScope ReAct 执行、ChatBI、 知识库、MCP 工具与嵌入式 Chat SDK。

  • 多专家路由、@ 提及直选与权限挂起恢复
  • 企业级 ChatBI、SQL 自愈与数据门户
  • 长期记忆、知识库、Trace 与调度中心

03

OpenClaw Buddy

为 OpenClaw 打造的零门槛 Web 管理台与对话伴侣，把 Bot、模型、Skills、渠道、 Workspace 文件和 Console 终端放进同一块工作台。

  • 可视化 Bot、模型、插件与渠道配置
  • Chat V3：聊天、文件、技能、终端同屏
  • 心跳探针、审计、自愈与飞书告警

Architecture

一条清晰的企业智能化链路。

底层治理数据，中层封装智能执行，上层面向业务系统、渠道和运营人员提供可控入口。

01 企业数据源

MySQL、ClickHouse、Oracle、业务库与非结构化文档。

02 数据服务化

元数据治理、SQL Lab、API 资源、目录发布与审计。

03 智能体执行

ChatBI、Knowledge、MCP、工具调用、任务调度与 Trace。

04 运营与渠道

嵌入式聊天、OpenClaw Buddy、飞书/微信/Telegram 等入口。

Experience

既能治理，也能让业务用户真正发现和使用数据。

数据产品目录、资产全景、权限申请与审批，让 API 不只是技术接口，而是可以被业务域浏览、 申请、追踪和运营的数据产品。

业务域目录 权限状态 调用趋势 审计回溯

Scenarios

高频企业场景，尽量做到开箱即用。

BI

自然语言查数

基于元数据、Few-Shot 案例、SQL 护栏与自愈链路，支撑企业级 ChatBI。

API

数据产品发布

把资源 API 包装为可发现、可申请、可审计的数据产品目录。

RAG

知识增强问答

面向非结构化文档的知识库管理、召回测试、引用注入与回答拦截。

MCP

工具生态连接

通过 MCP 与外部系统、插件、生产力工具连接，减少核心代码侵入。

SDK

业务系统嵌入

Embed Chat SDK 对接现有鉴权与租户体系，把智能体嵌到业务流程里。

OPS

Agent 运营工作台

OpenClaw Buddy 提供 Bot、模型、渠道、Workspace 与 Console 的可视化管理。

Delivery

面向私有化、可审计和可持续演进的交付方式。

三个项目都围绕企业内网、权限体系、多源数据和运维闭环设计，适合按模块分阶段落地： 先建数据服务底座，再接入智能体执行，最后补齐多渠道 Agent 运营入口。

私有化部署 Docker 打包、环境变量配置、离线部署流程。
安全与权限 RBAC、租户隔离、审计、脱敏、API Key 与会话鉴权。
可观测与追踪 调用统计、Trace 时间线、健康监控、日志与自愈事件。
